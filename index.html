AndroidUI起步For Android开发者

对Android开发者的AndroidUI框架起步教程.

代码使用TypeScript书写, 风格和语法与Java相似, 就算没有接触过TypeScript, 可以很快上手: http://www.typescriptlang.org/Handbook

1.准备环境

  1. 下载IDE: 支持Typescript的IDE或者编辑器. (推荐使用WebStorm)
  2. 安装Node.js: https://nodejs.org/
  3. 安装Typescript: npm install -g typescript
  4. 安装Babel转码器: npm install -g babel-cli
  5. 下载打开Hello World工程: 下载地址
  6. 如果使用WebStorm, 打开工程后, 需要在设置中开启TypeScript编译以获得更好代码提示和错误检查:
    Preferences -> Languages & Frameworks -> TypeScript: 1.Enable TypeScript Compiler. 2.use tsconfig.json

2.文件/目录说明

  1. app.html
    App入口页面, 可以修改’title’标签为你的App名字, ‘android-ui’标签里定义activity为App的入口Activity

  2. androidui-sdk
    AndroidUI框架存放的目录, 如果框架版本有更新, 需要升级替换其中的文件.

  3. build
    存放最终构建生成的js文件.

  4. res & gen
    res文件夹放图片,布局等资源, 在构建时会生成R文件在gen目录, 在代码引用R文件调用资源

  5. src
    代码的存放目录

如果使用WebStorm, 可以直接对app.html文件右键->Open In Browser 就可以查看.

  1. Java中可以省略this调用方法和属性, 但TypeScript中必须带上. 如: this.setContentView(…)

AndroidUI: 高性能WebApp框架

AndroidUI是一个高性能的WebApp框架, 使用Canvas渲染页面, 使得在移动浏览器上有原生App的体验.

为什么需要一个高性能的WebApp框架?

虽然目前WebApp的框架不少, 但是没有一个能够达到原生级别的流畅度和原生级别的体验.
制作一个高体验高性能的复杂交互, 对App来说较容易, 但是对移动浏览器来说, 非常困难.
现在用户已经习惯各种高质量和流畅度的原生App, 一个卡顿生硬的Web页面将很难被用户接受.
随着移动端设备越来越多, 微信应用号发布, 越来越多的页面需要被移动浏览器承载, 越来越复杂的交互, 要求越来越高的页面体验, 一个高性能的WebApp框架必须出现.

一个合格的WebApp框架需要做到什么

  1. 流畅度. 一个移动应用, 至少需要50fps才能让用户在交互中感受不到卡顿.
  2. 原生级别体验. 页面过渡动画, 滚动回弹, 点击响应等等细节都需要达到原生级别的体验.
  3. 稳定. 框架提供的接口和能力必须是通过测试, 可用于生产.
  4. 易用. 框架需要能够让开发者快速上手, 拥有丰富的组件, 能够快速制作出稳定可靠的页面.
  5. 开发者社区. 使用框架中问题能够快速搜索到答案.

为什么是AndroidUI?

AndroidUI使用Canvas绘制界面.
所有组件从Android平台移植而来, 与原生AndroidApp一致的体验和稳定.
对Android开发者来说可以轻易上手, 对Web开发者来说, 也是与普通页面相似的开发方式(自定义的HTML标签+JS)
因为移植, 框架与AndroidSDK的API一致, 所有API文档用法和问题等都可以从Android社区找到.

使用Canvas渲染界面

DOM性能低下, 普通HTML5页面不能流畅展现复杂界面, 纯Canvas来渲染界面是一种有效方案.

移植自Android

Android是开源的, Android的组件是由Java书写, AndroidUI框架做了视图底层的一套实现, Android现有的组件都可以基于此移植到Web端.

优秀稳定易用的组件

与AndroidApp开发定义页面的方式一致, AndroidUI使用组件来构造一个页面. 使用Html代码来声明页面布局, Html代码中的一种标签代表一种组件.
Android核心的组件都已经移植到AndroidUI, 同时AndroidUI还提供了一些专属的组件.

单页面应用

AndroidUI拥有一个优秀的页面栈控制(页面路由)的实现.
页面栈和页面数据能够在浏览器刷新后保持.
页面能够响应浏览器的后退而关闭, 还能’截获’后退的点击并实现自定义逻辑.
